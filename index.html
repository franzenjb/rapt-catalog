How this was built

A plain-English guide for the RAPT / Experience Builder team. No magic — it’s one web page that reads RAPT’s own layers.

1
Describe it to AI

Plain English: “Build a searchable catalog of RAPT’s layers.” The AI writes one index.html.

2
Open it

Double-click the file — it runs in any browser. No server, no build step.

3
Host where gov trusts

Your ArcGIS Online / Enterprise or cloud.gov (FedRAMP) — already authorized.

4
Embed in Experience Builder

Add the Embed widget, paste the URL. Done.

  • One HTML file. No server, no database, no build.
  • The 106 layers are RAPT’s real web-map layers, read straight from ArcGIS REST.
  • Search, filters, and the layouts are plain browser JavaScript — the same web tech Experience Builder runs on.
  • A — Custom EB widget (best for your team). EB widgets are React. Wrap this same logic as a widget — it runs natively inside your EB app, no outside hosting, no new approval. Ships with the app to ArcGIS.
  • B — Hosted page + Embed widget. Keep it as one HTML page, host it (below), and point EB’s Embed widget at the URL. Fastest way to reuse exactly what you see here.
  • Already authorized: publish inside the ArcGIS Enterprise / ArcGIS Online you already run — it carries your agency’s authorization.
  • Static page: cloud.gov / cloud.gov Pages (GSA, FedRAMP), or AWS GovCloud / Azure Government under your existing ATO.
  • Not for production: Vercel / Netlify / GitHub Pages are perfect for a quick prototype (like this demo) — not for an ATO’d production app.
  • Authorized host — FedRAMP or an existing ATO (ArcGIS Enterprise, cloud.gov, GovCloud).
  • 508 accessible — keyboard navigation, color contrast (WCAG AA), labels.
  • No secrets in the browser — public/read-only services or a platform token; never credentials in the HTML.
  • Authoritative data — the real RAPT services, no copies.

The whole idea in ~10 lines — a searchable layer list:

<input placeholder="Search layers" oninput="f(this.value)">
<ul id="list"></ul>
<script>
  const layers = ["Flood Hazard","Hospitals","Census Tracts"]; // from RAPT
  function f(q){
    list.innerHTML = layers
      .filter(n => n.toLowerCase().includes(q.toLowerCase()))
      .map(n => "<li>"+n+"</li>").join("");
  }
  f("");
</script>

Then in Experience Builder: Insert → Embed → paste your hosted URL. That’s the whole pipeline.

This live demo is a prototype on a commercial host to show the idea fast. The production version belongs inside your authorized ArcGIS / cloud.gov environment.
